<h1 align="center">Getting Support</h1>

Need help? This guide explains the best way to get assistance across **The Lupaxa Project**.

Using the correct support channel helps us respond more quickly and ensures your request reaches the people best placed to help.

## Quick Reference

| I want to...                     | Where should I go?                                                      |
| :------------------------------- | :---------------------------------------------------------------------- |
| Report a bug                     | Open a **Bug Report** issue.                                            |
| Suggest a new feature            | Open a **Feature Request** issue.                                       |
| Ask a question                   | Use **GitHub Discussions** (where available) or a **Question** issue.   |
| Report a security vulnerability  | Follow the **Security Policy** and contact the security team privately. |
| Contribute code or documentation | Read the **How to Contribute** guide                                    |

> [!TIP]
> Before opening a new issue, please search existing issues and discussions. Your question or problem may already have been answered.

## Reporting Bugs

If you've discovered a bug, please open a **Bug Report** using the repository's issue templates where available.

Including the following information will help us investigate more quickly:

- Steps to reproduce the issue
- Expected behaviour
- Actual behaviour
- Operating system and environment
- Tool version or Git commit (where applicable)
- Relevant logs, screenshots, or error messages

The easier a problem is to reproduce, the easier it is to fix.

## Requesting Features

Feature requests are always welcome.

When submitting a request, please describe:

- The problem you're trying to solve
- Why the feature would be valuable
- Any alternative approaches you've considered
- Examples or mock-ups where appropriate

We review every suggestion, although not every request will become part of the project.

## Asking Questions

If you're unsure how something works, need help getting started, or would like clarification on expected behaviour, please ask.

Where available, GitHub Discussions are the preferred place for general questions. If Discussions are not enabled, use the repository's **Question**
issue template.

## Reporting Security Issues

> [!IMPORTANT]
> Please **do not** report security vulnerabilities through public issues, pull requests, or discussions.

Instead, follow the responsible disclosure process described in the organisation-wide **Security Policy**.

Responsible disclosure helps protect users while a fix is investigated and prepared.

## Response Times

**The Lupaxa Project** is maintained by volunteers.

Although we aim to respond as quickly as possible, response times will vary depending on:

- Maintainer availability
- Project priorities
- The complexity of the request
- The information provided

We appreciate your patience and will always do our best to help.
