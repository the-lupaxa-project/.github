<h1 align="center">Projects Overview</h1>

The Lupaxa Project is organised into multiple GitHub organisations, each focused on a particular area of technology or engineering.

Rather than maintaining a single, large organisation, related repositories are grouped together to improve discoverability, simplify management, and allow
each area of the project to evolve independently while remaining part of the wider Lupaxa ecosystem.

Each organisation shares the same community standards, contribution guidelines, and engineering principles, providing a consistent experience for contributors
regardless of which repositories they work with.

## Shared Standards

Although each organisation focuses on a different area, all repositories across **The Lupaxa Project** share a common set of standards covering:

- Community participation
- Contributing guidelines
- Governance
- Repository maintenance
- Security practices
- Documentation quality

This consistency allows contributors to move between organisations with minimal friction while allowing each project to retain its own technical identity.

## Growing the Ecosystem

The Lupaxa Project continues to evolve.

As new technologies emerge and new ideas are explored, additional organisations may be created where they provide a clear benefit to contributors and users.
Our goal is to keep projects logically organised while maintaining a consistent experience across the ecosystem.
